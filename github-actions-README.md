# Звіт: Secure SDLC Pipeline для OWASP Juice Shop

## 1. Створення CI/CD пайплайну

Для проекту OWASP Juice Shop створено GitHub Actions workflow (`secure-pipeline.yml`), який автоматично запускається при push та pull_request. Пайплайн послідовно виконує: лінтинг Dockerfile (Hadolint), збірку Docker-образу, сканування на вразливості (Snyk), аналіз якості коду (SonarQube) та перевірку структури образу (Dockle).

![Створення workflow файлу]()

## 2. Налаштування секретів

Для коректної роботи інтеграцій додано секрети в налаштуваннях репозиторію: `SNYK_TOKEN` для сканування залежностей.

![Додавання секретів](2.png)

## 3. Збірка Docker-образу Juice Shop

Пайплайн виконує збірку Docker-образу на базі офіційного Juice Shop. Використовується multi-stage build для оптимізації розміру фінального образу.

![Збірка Docker-образу](3.png)

## 4. Сканування Snyk — аналіз залежностей

Snyk автоматично сканує npm-залежності проекту та Docker-образ на відомі вразливості. Результати зберігаються як артефакт пайплайну.

![Результати Snyk](4.png)

## 5. Аналіз SonarQube — якість коду

SonarQube виконує статичний аналіз кодової бази: перевіряє code smells, баги, дублювання та покриття тестами. Дашборд відображає загальну оцінку проекту.

![Дашборд SonarQube](5.png)

## 6. Сканування Dockle — перевірка Docker-образу

Dockle аналізує зібраний образ на відповідність best practices безпеки контейнерів (CIS Benchmarks). Додано `continue-on-error: true` щоб пайплайн генерував повний звіт навіть при знайдених проблемах.

![Результати Dockle](6.png)

## 7. Фінальний звіт GitHub Actions

Всі етапи пайплайну відпрацювали. Загальний статус виконання та артефакти доступні у вкладці Actions.

![Фінальний звіт](7.png)

## 8. Завантаження звітів

З артефактів завантажено згенеровані файли: `snyk-report.json`, `sonar-report.json` та `dockle-report.json`.

![Завантаження артефактів](8.png)

## Знайдені вразливості

### Critical

**Prototype Pollution (lodash)**
- **Опис:** Вразливість Prototype Pollution у бібліотеці lodash дозволяє зловмиснику модифікувати прототип Object через спеціально сформовані вхідні дані.
- **Встановлена версія:** 4.17.15
- **Виправлена версія:** 4.17.21

### High

**Remote Code Execution (express-jwt)**
- **Опис:** Відсутність валідації алгоритму підпису JWT-токенів дозволяє підробити токен автентифікації.
- **Встановлена версія:** 0.1.3
- **Виправлена версія:** 6.0.0

**SQL Injection (sequelize)**
- **Опис:** Некоректна обробка спеціальних символів у SQL-запитах може призвести до несанкціонованого доступу до бази даних.
- **Встановлена версія:** 5.21.3
- **Виправлена версія:** 5.22.5

### Medium

**Cross-Site Scripting — XSS (sanitize-html)**
- **Опис:** Обхід санітизації HTML дозволяє впровадити шкідливий JavaScript-код через спеціально сформований вхідний рядок.
- **Встановлена версія:** 1.20.0
- **Виправлена версія:** 2.3.2
